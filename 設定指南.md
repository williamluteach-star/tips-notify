# 詳細設定指南

## 第一步：申請LINE Developers帳號

### 1.1 建立Provider
1. 前往 [LINE Developers](https://developers.line.biz/)
2. 登入您的LINE帳號
3. 點擊「Create」建立新的Provider
4. 輸入Provider名稱（例如：您的機構名稱）

### 1.2 建立Messaging API Channel
1. 在Provider下點擊「Create」
2. 選擇「Messaging API」
3. 填寫Channel資訊：
   - **Channel name**: 例如「作業通知系統」
   - **Channel description**: 描述用途
   - **Category**: 選擇「Education」
   - **Subcategory**: 選擇合適的分類
   - **Email address**: 您的聯絡信箱
4. 同意條款並建立

### 1.3 取得憑證
1. 進入Channel設定頁面
2. 在「Messaging API」標籤下找到：
   - **Channel access token**: 複製此token
   - **Channel secret**: 複製此secret
3. 將這些值填入 `.env` 檔案

### 1.4 設定Webhook
1. 在「Messaging API」標籤下找到「Webhook settings」
2. 輸入您的Webhook URL（部署後才能設定）
3. 啟用「Use webhook」

## 第二步：設定Google Cloud專案

### 2.1 建立專案
1. 前往 [Google Cloud Console](https://console.cloud.google.com/)
2. 點擊專案選擇器，選擇「New Project」
3. 輸入專案名稱（例如：homework-notification）
4. 建立專案

### 2.2 啟用Google Sheets API
1. 在專案中，前往「APIs & Services」>「Library」
2. 搜尋「Google Sheets API」
3. 點擊「Enable」啟用

### 2.3 建立服務帳號
1. 前往「APIs & Services」>「Credentials」
2. 點擊「Create Credentials」>「Service Account」
3. 輸入服務帳號名稱（例如：homework-bot）
4. 點擊「Create and Continue」
5. 選擇角色：選擇「Editor」或「Owner」（僅用於測試，生產環境建議更嚴格）
6. 點擊「Done」

### 2.4 建立並下載憑證
1. 在服務帳號列表中，點擊剛建立的服務帳號
2. 前往「Keys」標籤
3. 點擊「Add Key」>「Create new key」
4. 選擇「JSON」格式
5. 下載JSON檔案

### 2.5 從JSON取得憑證資訊
開啟下載的JSON檔案，您會看到：
```json
{
  "type": "service_account",
  "project_id": "...",
  "private_key_id": "...",
  "private_key": "-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n",
  "client_email": "...@....iam.gserviceaccount.com",
  ...
}
```

將以下值填入 `.env`：
- `GOOGLE_SERVICE_ACCOUNT_EMAIL`: `client_email` 的值
- `GOOGLE_PRIVATE_KEY`: `private_key` 的值（包含 `-----BEGIN PRIVATE KEY-----` 和 `-----END PRIVATE KEY-----`）

## 第三步：建立Google Sheets

### 3.1 建立新的試算表
1. 前往 [Google Sheets](https://sheets.google.com/)
2. 建立新的試算表
3. 命名為「作業通知系統」或您喜歡的名稱

### 3.2 建立工作表
建立三個工作表：

#### 工作表1：學生資料表
在第一行輸入標題：
| A | B | C | D | E | F |
|---|---|---|---|---|---|
| 學生姓名 | 年級 | 家長LINE ID | 家長姓名 | 聯絡電話 | 備註 |

#### 工作表2：作業記錄表
在第一行輸入標題：
| A | B | C | D | E | F | G |
|---|---|---|---|---|---|---|
| 時間戳記 | 學生姓名 | 作業項目 | 完成時間 | 操作人員 | 通知狀態 | 備註 |

#### 工作表3：作業模板表（可選）
| A | B | C | D |
|---|---|---|---|
| 作業類別 | 作業名稱 | 適用年級 | 預設訊息模板 |

### 3.3 取得Sheets ID
從網址中取得Sheets ID：
```
https://docs.google.com/spreadsheets/d/XXXXX/edit
                                    ^^^^^
                                    這就是Sheets ID
```

將此ID填入 `.env` 的 `GOOGLE_SHEETS_ID`

### 3.4 授予服務帳號權限
1. 在Google Sheets中，點擊右上角「共用」
2. 輸入服務帳號的email（`client_email`）
3. 選擇「編輯者」權限
4. 點擊「傳送」

## 第四步：取得家長LINE User ID

### 方法一：透過Webhook事件（推薦）
1. 部署系統並設定Webhook
2. 請家長加LINE官方帳號為好友
3. 當家長加好友時，系統會收到 `follow` 事件
4. 在伺服器日誌中查看 `event.source.userId`，這就是LINE User ID
5. 將此ID填入「學生資料表」的「家長LINE ID」欄位

### 方法二：使用LINE官方帳號後台
1. 在LINE Developers Console中，前往「Messaging API」>「Webhook settings」
2. 啟用「Webhook」並設定URL
3. 當家長發送訊息時，可以在Webhook事件中取得User ID

### 方法三：使用LINE官方帳號管理後台
1. 登入LINE官方帳號管理後台
2. 在「好友」列表中可以看到已加好友的使用者
3. 點擊使用者可以查看詳細資訊（包含User ID）

## 第五步：測試系統

### 5.1 執行連線測試
```bash
node scripts/test-connection.js
```

### 5.2 測試記錄作業
使用Postman或curl測試API：
```bash
curl -X POST http://localhost:3000/api/homework \
  -H "Content-Type: application/json" \
  -d '{
    "studentName": "測試學生",
    "homeworkItem": "測試作業",
    "operator": "測試人員"
  }'
```

### 5.3 檢查通知
確認家長是否收到LINE通知訊息

## 常見問題排除

### 問題1：LINE Bot無法發送訊息
- 檢查Channel Access Token是否正確
- 確認家長已加官方帳號為好友
- 檢查訊息額度是否用完

### 問題2：Google Sheets無法讀寫
- 確認服務帳號有編輯權限
- 檢查Sheets ID是否正確
- 確認工作表名稱是否正確（「學生資料表」、「作業記錄表」）

### 問題3：找不到學生資料
- 確認「學生資料表」中的學生姓名與記錄時輸入的完全一致
- 檢查是否有空白或特殊字元

### 問題4：Webhook無法接收訊息
- 確認Webhook URL可以從外網訪問
- 檢查HTTPS設定（LINE要求HTTPS）
- 確認伺服器正在運行
