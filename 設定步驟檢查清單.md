# 設定步驟檢查清單

## ✅ 完成狀態追蹤

### 第一步：LINE Bot 設定

- [ ] 1.1 登入 LINE Developers Console
  - 網址：https://developers.line.biz/console/
  - 使用您現有的帳號登入

- [ ] 1.2 選擇或建立 Provider
  - 如果已有 Provider，直接選擇
  - 如果沒有，建立新的 Provider

- [ ] 1.3 選擇或建立 Messaging API Channel
  - 如果已有 Channel，直接使用
  - 如果沒有，建立新的 Messaging API Channel

- [ ] 1.4 取得憑證
  - [ ] Channel Access Token：`_________________________`
  - [ ] Channel Secret：`_________________________`
  - 位置：Channel 設定頁面 > Messaging API 標籤

- [ ] 1.5 填入 .env 檔案
  - 使用設定助手：`node scripts/setup-helper.js`
  - 或手動編輯 .env 檔案

---

### 第二步：Google Cloud 設定

- [ ] 2.1 建立或選擇專案
  - 網址：https://console.cloud.google.com/
  - 專案名稱：`_________________________`

- [ ] 2.2 啟用 Google Sheets API
  - 前往：APIs & Services > Library
  - 搜尋「Google Sheets API」
  - 點擊「Enable」

- [ ] 2.3 建立服務帳號
  - 前往：APIs & Services > Credentials
  - 點擊「Create Credentials」>「Service Account」
  - 服務帳號名稱：`_________________________`
  - 角色：選擇「Editor」或「Owner」（測試用）

- [ ] 2.4 下載 JSON 憑證
  - 在服務帳號列表中，點擊剛建立的帳號
  - 前往「Keys」標籤
  - 點擊「Add Key」>「Create new key」
  - 選擇「JSON」格式並下載

- [ ] 2.5 從 JSON 取得資訊
  - 開啟下載的 JSON 檔案
  - 複製 `client_email` 的值：`_________________________`
  - 複製 `private_key` 的值（完整內容，包含 BEGIN/END 標記）

- [ ] 2.6 填入 .env 檔案
  - GOOGLE_SERVICE_ACCOUNT_EMAIL：`_________________________`
  - GOOGLE_PRIVATE_KEY：`_________________________`

---

### 第三步：Google Sheets 設定

- [ ] 3.1 建立新的試算表
  - 網址：https://sheets.google.com/
  - 建立新的試算表
  - 命名為：`_________________________`

- [ ] 3.2 取得 Sheets ID
  - 從網址中取得 ID
  - 網址格式：`https://docs.google.com/spreadsheets/d/XXXXX/edit`
  - Sheets ID：`_________________________`
  - 填入 .env 的 `GOOGLE_SHEETS_ID`

- [ ] 3.3 建立工作表
  - [ ] 工作表1：命名為「學生資料表」
    - 在第一行輸入標題：
      | 學生姓名 | 年級 | 家長LINE ID | 家長姓名 | 聯絡電話 | 備註 |
  
  - [ ] 工作表2：命名為「作業記錄表」
    - 在第一行輸入標題：
      | 時間戳記 | 學生姓名 | 作業項目 | 完成時間 | 操作人員 | 通知狀態 | 備註 |
  
  - [ ] 工作表3（可選）：命名為「作業模板表」
    - 在第一行輸入標題：
      | 作業類別 | 作業名稱 | 適用年級 | 預設訊息模板 |

- [ ] 3.4 授予服務帳號權限
  - 在 Google Sheets 中，點擊右上角「共用」
  - 輸入服務帳號的 email（步驟 2.5 的 client_email）
  - 選擇「編輯者」權限
  - 點擊「傳送」

---

### 第四步：測試連線

- [ ] 4.1 執行連線測試
  ```bash
  node scripts/test-connection.js
  ```

- [ ] 4.2 檢查測試結果
  - [ ] LINE Bot 連線：✅ 成功
  - [ ] Google Sheets 連線：✅ 成功

- [ ] 4.3 測試記錄作業
  - 在 Web 介面中測試記錄一筆作業
  - 檢查 Google Sheets 是否有新記錄

---

### 第五步：取得家長 LINE User ID

- [ ] 5.1 設定 Webhook（部署後）
  - 在 LINE Developers Console 設定 Webhook URL
  - 啟用 Webhook

- [ ] 5.2 請家長加好友
  - 請家長加 LINE 官方帳號為好友
  - 系統會收到 `follow` 事件

- [ ] 5.3 取得 User ID
  - 從 Webhook 事件中取得 `event.source.userId`
  - 或從 LINE 官方帳號管理後台查看

- [ ] 5.4 填入學生資料表
  - 在「學生資料表」中輸入學生資訊
  - 將 User ID 填入「家長LINE ID」欄位

---

## 📝 快速參考

### .env 檔案格式
```env
LINE_CHANNEL_ACCESS_TOKEN=你的token
LINE_CHANNEL_SECRET=你的secret
GOOGLE_SHEETS_ID=你的sheets_id
GOOGLE_SERVICE_ACCOUNT_EMAIL=你的service_account_email
GOOGLE_PRIVATE_KEY=你的private_key
PORT=3000
NODE_ENV=development
```

### 常用指令
```bash
# 啟動伺服器
npm start

# 測試連線
node scripts/test-connection.js

# 匯入學生資料
node scripts/import-students.js scripts/example-students.csv
```
